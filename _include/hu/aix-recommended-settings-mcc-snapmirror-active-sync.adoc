.MetroCluster
--
預設情況下，當沒有可用的 LUN 路徑時，AIX OS 會強制執行較短的 I/O 逾時。這可能發生在包含單一交換器 SAN 結構的配置中以及經歷計劃外故障轉移的MetroCluster配置中。有關其他資訊和建議對預設設定進行更改，請參閱知識庫文章link:https://kb.netapp.com/on-prem/ontap/mc/MC-KBs/What_are_AIX_Host_support_considerations_in_a_MetroCluster_configuration["MetroCluster配置中的 AIX 主機支援注意事項有哪些？"^]。

--
.SnapMirror主動同步
--
AIX 主機支援 SnapMirror 主動同步。不同 ONTAP 版本的支援行為有所不同：

** 從 ONTAP 9.19.1 開始，Active/Active 組態支援 SnapMirror 主動式同步，可實現不中斷營運的 SnapMirror 主動式同步作業。請參閱知識庫文章 link:https://kb.netapp.com/on-prem/ontap/DP/SnapMirror/SnapMirror-KBs/SnapMirror_active_sync_support_for_AIX%3A_behavior_by_ONTAP_release_and_migration_guidance["SnapMirror 主動同步對 AIX 的支援：依 ONTAP 版本的行為與遷移指南"^]。
** 對於 ONTAP 9.11.1 至 9.18.1 版本，SnapMirror 主動同步在主動/待命配置中受到支援。請參閱知識庫文章 link:https://kb.netapp.com/on-prem/ontap/DP/SnapMirror/SnapMirror-KBs/How_to_configure_AIX_Host_for_SnapMirror_active_sync_in_ONTAP["如何在 ONTAP 9.11.1 至 9.18.1 版本中設定 AIX 主機以進行 SnapMirror 活動同步"^]。
