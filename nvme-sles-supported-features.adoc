= 了解 SUSE Linux Enterprise Server 對ONTAP 的支援與功能
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
使用 NVMe over Fabrics (NVMe-oF) 進行主機配置所支援的功能因ONTAP和 SUSE Linux Enterprise Server 的版本而異。

[cols="50,25,25"]
|===
| 特徵 | SUSE Linux Enterprise Server 主機版本 | ONTAP 版本 


| NVMe/TCP 報告 ASA r2 系統的所有最佳化路徑  a| 
16
| 9.16.1 或更高版本 


| NVMe/TCP 支援傳輸層安全協定 (TLS) 1.3 加密。  a| 
15 SP6 或更高版本
| 9.16.1 或更高版本 


| RHEL 主機與ONTAP控制器之間透過 NVMe/TCP 支援安全的帶內驗證。  a| 
15 SP4 或更高版本
| 9.12.1 或更高版本 


| 持久發現控制器 (PDC) 使用唯一的發現 NQN 提供支持  a| 
15 SP4 或更高版本
| 9.11.1 或更高版本 


| NVMe/TCP 使用原生命名空間提供命名空間 `nvme-cli`包裹  a| 
15 SP4 或更高版本
| 9.10.1 或更高版本 


| 在同一主機上支援 NVMe 和 SCSI 流量，NVMe-oF 命名空間使用 NVMe 多路徑，SCSI LUN 使用 dm 多路徑。  a| 
15 SP1 或更高版本
| 9.4 或更高版本 
|===
無論系統設定中執行的ONTAP版本為何， ONTAP支援以下 SAN 主機功能。

[cols="75,25"]
|===
| 特徵 | SUSE Linux Enterprise Server 主機版本 


| SAN 啟動已透過 NVMe/FC 協定啟用  a| 
15 SP7 或更高版本



| 預設啟用原生 NVMe 多路徑。  a| 
15 SP1 或更高版本



| 這 `nvme-cli`軟體包包含自動連接腳本，無需第三方腳本。  a| 
15 SP1 或更高版本

|===

NOTE: 有關受支援配置的詳細信息，請參閱link:https://mysupport.netapp.com/matrix/["互通性對照表工具"^]。
