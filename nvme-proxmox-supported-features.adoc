= 了解 Proxmox 主機對ONTAP 的支援與功能
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
使用 NVMe over Fabrics (NVMe-oF) 進行主機配置所支援的功能因ONTAP和 Proxmox 的版本而異。

[cols="30,20,20"]
|===
| ONTAP功能 | Proxmox主機版本 | ONTAP 版本 


| NVMe/TCP 是一項完全支援的企業級功能  a| 
9.0
| 9.10.1 或更高版本 


| 使用原生功能提供 NVMe/FC 和 NVMe/TCP 命名空間的ONTAP詳細信息 `nvme-cli`包裹。  a| 
8.0
| 9.10.1 或更高版本 


| 在同一主機上支援 NVMe 和 SCSI 流量，NVMe-oF 命名空間使用 NVMe 多路徑，SCSI LUN 使用 dm 多路徑。  a| 
8.0
| 9.4 或更高版本 
|===
無論系統設定中執行的ONTAP版本為何， ONTAP支援以下 SAN 主機功能。

[cols="70,30"]
|===
| 特徵 | Proxmox主機版本 


| 這 `nvme-cli`軟體包包含自動連接腳本，無需第三方腳本。  a| 
9.0



|  `nvme-cli` 軟體套件中的原生 udev 規則為 NVMe 多路徑提供輪詢負載平衡  a| 
9.0



| 預設啟用原生 NVMe 多路徑。  a| 
8.0

|===

NOTE: 有關受支援配置的詳細信息，請參閱link:https://mysupport.netapp.com/matrix/["互通性對照表工具"^]。
